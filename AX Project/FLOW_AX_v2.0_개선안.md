## 개선 1: Phase -1 신설 — 데이터 레디니스 진단

### 왜 필요한가

Gartner 2025 보고서에 따르면, 저성숙 조직의 34%가 **데이터 가용성과 품질 문제**를 AI 구현의 최대 장애물로 지목합니다. 현재 v1.0은 고객사의 데이터 성숙도를 전제하지 않아, Phase 1 RTC 캔버스의 데이터 신뢰성이 보장되지 않습니다.

### v2.0 설계

**기간:** 1주 (Phase 0 이전)

**진단 영역 — 5축 평가 (각 20점, 총 100점)**

| 축 | 평가 항목 | 참고 프레임워크 |
|----|----------|---------------|
| 데이터 인프라 | 통합 데이터 플랫폼 존재 여부, 클라우드/온프레 구조 | CMMI DMM 5단계 |
| 데이터 품질 | 정확성, 완전성, 일관성, 적시성 | Gartner 7영역 |
| 거버넌스 체계 | 소유권 명확화, 메타데이터 관리, 접근 권한 | ISO/IEC 42001 |
| 기술 역량 | 데이터 엔지니어 유무, 분석 도구 활용 수준 | MITRE AI Maturity |
| 조직 문화 | 데이터 기반 의사결정 습관, 실험 친화도 | Gartner 문화 영역 |

**판정 기준:**

| 점수 | 판정 | 조치 |
|------|------|------|
| 80점 이상 | AX Ready | Phase 0 즉시 진입 |
| 60~79점 | 조건부 Ready | 부족 영역 보완 병행하며 Phase 0 진입 |
| 40~59점 | Not Ready | 데이터 인프라 구축 선행 (2~3개월) 후 재진단 |
| 39점 이하 | 재검토 | DX 기초부터 재설계 필요 |

**산출물:** 데이터 레디니스 진단 리포트, 보완 로드맵, Phase 0 진입 판단서

**데미안의 고유 무기:** 이 진단 자체를 "유료 진단 서비스"로 패키징 가능. 진단 결과가 좋으면 본 계약으로, 부족하면 데이터 기초 컨설팅으로 연결.

## 개선 2: Phase 3 업그레이드 — Agentic AI 시대 업무 재설계

### 왜 필요한가

Gartner 예측에 따르면 2026년 엔터프라이즈 앱의 40%가 업무 전용 AI 에이전트를 포함할 전망입니다. McKinsey에 따르면 업무 재설계를 수행한 고성과 기업이 동료 대비 3배 높은 성과를 냅니다. v1.0의 "AI 초안 생성 → 인간 검수" 모델은 이미 2024년 패러다임이며, 2026년에는 부족합니다.

### v2.0 설계 — T3 업무 분류 확장 (3단계 → 5단계)

**기존 v1.0:**
1. 완전 자동화 가능
2. AI 보조 + 인간 감독
3. 인간 전용

**v2.0 확장:**
1. **AI 에이전트 자율 실행** — 인간은 목표·제약조건만 설정, 에이전트가 의사결정·실행·예외처리까지 자율 수행
2. **AI 에이전트 오케스트레이션** — 복수 에이전트가 협업하는 워크플로우, 인간은 전체 흐름 설계·감독
3. **AI 보조 + 인간 감독** — AI가 초안 생성, 인간이 검수·수정 (기존 유지)
4. **인간 주도 + AI 인사이트** — 인간이 판단하되 AI가 데이터 분석·옵션 제시
5. **인간 전용** — 관계·윤리·창의적 판단 (기존 유지)

### Phase 3 워크숍 업그레이드 — WHY-WHAT-HOW + AGENT 세션 추가

**AGENT 세션 (신규, 2시간)**

1. 에이전트 적합성 평가: 어떤 업무가 에이전트 자율 실행에 적합한가?
   - 판단 기준: 규칙 명확성, 오류 허용 범위, 피드백 루프 존재 여부
2. 에이전트 아키텍처 설계: 단일 에이전트 vs 멀티 에이전트 vs 인간-에이전트 하이브리드
3. 인간-에이전트 핸드오프 프로토콜: 에이전트가 "확신이 낮을 때" 인간에게 넘기는 기준
4. 에이전트 신뢰도 메트릭 설계: 정확도, 컴플라이언스 준수율, 예외 처리 성공률

**직군별 에이전트 시나리오 추가:**

| 직군 | 에이전트 시나리오 예시 |
|------|---------------------|
| 마케팅 | 콘텐츠 기획→초안→이미지→교정→발행→성과분석을 멀티 에이전트가 자율 실행 |
| 재무 | 데이터 수집→이상치 탐지→보고서 초안→검토 요청까지 에이전트 자동화 |
| 인사 | 채용 서류 1차 스크리닝→면접 일정 조율→온보딩 자료 개인화를 에이전트 수행 |
| 영업 | 리드 스코어링→개인화 제안서 생성→후속 이메일 자동 발송→CRM 업데이트 |
| 고객지원 | 문의 분류→1차 응대→에스컬레이션 판단→사후 만족도 조사까지 에이전트 |
| R&D | 논문 검색→요약→실험 설계 초안→데이터 분석 스크립트 생성 |

## 개선 3: Phase 4 확장 — 중간관리자 전환 모델

### 왜 필요한가

HBR 2025 연구에 따르면, 관리자 6명 중 4명이 AI로 자동화 가능한 행정 업무에 절반 이상의 시간을 소비합니다. Deloitte는 2030년까지 미국 관리직 150만 개가 위험하다고 예측합니다. 대기업 AX에서 가장 큰 저항은 중간관리자에게서 옵니다 — 이들의 존재 이유인 "정보 중계"와 "업무 분배"를 AI가 대체하기 때문입니다.

### v2.0 설계 — 중간관리자 3트랙 전환 모델

**트랙 A: 코치/전략가 전환 (대상의 약 70%)**
- 현재 역할: 업무 배분, 진행 관리, 보고서 취합
- 전환 역할: 팀 성장 코칭, AI 에이전트 거버넌스, 전략 의사결정
- 필요 역량: 코칭 스킬, AI 리터러시, 데이터 스토리텔링
- 지원: STAR-AR 코칭 + AI 코칭 플랫폼 활용 교육
- KPI 전환: "관리한 업무 건수" → "팀원 역량 성장도 + 전략 기여도"

**트랙 B: AI 리드/워크플로우 설계자 (대상의 약 20%)**
- 현재 역할: 기술적 관리, 프로세스 최적화
- 전환 역할: 에이전트 워크플로우 설계, AI 성능 최적화
- 필요 역량: 고급 AI 활용, 프로세스 마이닝, 자동화 설계
- 지원: AI 심화 교육 3~6개월 집중 과정
- KPI 전환: "프로세스 관리" → "자동화율 + 에이전트 신뢰도"

**트랙 C: 리스크/컴플라이언스 감독 (대상의 약 10%)**
- 현재 역할: 품질 관리, 규정 준수 감독
- 전환 역할: AI 거버넌스, 윤리적 AI 운영 감독, 감사
- 필요 역량: NIST AI RMF 이해, ISO 42001, 법규 지식
- 지원: 거버넌스 전문 교육
- KPI 전환: "규정 준수율" → "AI 리스크 관리 성과 + 감사 품질"

**진행 방법:**
1. Phase 1 RTC 캔버스의 R3(성장 벡터)에 "중간관리자 전환 트랙" 항목 추가
2. Phase 2 ICEP에서 P축 평가 시 "팀장의 전환 준비도"를 별도 세부 항목으로 분리
3. Phase 4 파일럿에서 중간관리자 대상 "전환 코칭 프로그램" 병행 실행

## 개선 4: Phase 5 강화 — AI 거버넌스 국제 표준 연결

### 왜 필요한가

대기업 HRD 제안 시 거버넌스는 필수 항목입니다. 특히 EU AI Act(2024.8 발효), 한국 AI 기본법 논의, ISO/IEC 42001 인증 확산 맥락에서, "4영역 선언" 수준으로는 신뢰를 얻기 어렵습니다.

### v2.0 설계 — NIST AI RMF 4함수 + FLOW~ 거버넌스 통합

| NIST 4함수 | FLOW~ 기존 4영역 | v2.0 통합 |
|-----------|----------------|----------|
| GOVERN (거버넌스) | 책임 정책 | AI 거버넌스 위원회 구성, 역할·책임 정의, 정기 리뷰 체계 |
| MAP (매핑) | 데이터 정책 | AI 시스템별 위험 등급 분류, 데이터 등급별 AI 입력 허용 기준, 영향 범위 사전 검토 |
| MEASURE (측정) | 품질 정책 | AI 출력물 정확도·공정성·설명가능성 정기 측정, 편향 모니터링, EARS와 연동 |
| MANAGE (관리) | 도구 정책 | 인시던트 대응 플레이북, 자동 롤백 기준, 도구 승인·폐기 프로세스 |

**추가 요소:**
- EU AI Act 위험 분류 매핑 (EU 역내 운영 기업 대상)
- 한국 개인정보보호법(PIPA) AI 관련 조항 반영
- ISO/IEC 42001 인증 로드맵 (3~5년 계획, 선택)
- 감사 로그 자동화: 모든 AI 의사결정의 추적 가능성 확보

## 개선 5: 전 Phase 관통 — 4기둥 ROI 계산 모델

### 왜 필요한가

IBM 보고서에 따르면 CEO의 29%만 AI ROI를 자신감 있게 측정 가능합니다. Centage 연구에 따르면 약 25%의 AI 이니셔티브만 예상 ROI를 달성합니다. ROI 모델 없이는 대기업 예산 품의가 불가능합니다.

### v2.0 설계 — FLOW~ AX ROI 4기둥 모델

**기본 공식:**
```
AX ROI(%) = (순편익 - 총투자액) / 총투자액 × 100

순편익 = 효율성 절감 + 수익 증대 + 리스크 회피 + 민첩성 가치
```

**4기둥 상세:**

| 기둥 | 가중치 | 측정 항목 | 산출 방법 |
|------|--------|----------|----------|
| 효율성 (Efficiency) | 40~50% | 처리 시간 단축, FTE 절감, 초과근무 감소 | 자동화 대상 FTE × 평균 연봉 + 생산성 향상% × 영향 FTE × 연봉 |
| 수익 증대 (Revenue) | 20~25% | 거래 속도 향상, 고객 만족도, 크로스셀 | AI 도입 후 추가 매출 × 이익률 |
| 리스크 완화 (Risk) | 20~25% | 컴플라이언스 위반 감소, 오류율 감소 | 위반 확률 감소분 × 평균 벌금/손실액 |
| 민첩성 (Agility) | 10~15% | 의사결정 속도, 시장 반응 시간, 이탈률 | 시간 단축 × 기회 비용 + 이탈률 감소 × 재교육비 |

**Phase별 ROI 측정 시점:**

| Phase | ROI 활동 |
|-------|---------|
| Phase -1 | Baseline 데이터 수집 (현재 비용·시간·오류율) |
| Phase 2 | ICEP 점수와 ROI 예측치 연동 (업무별 예상 ROI) |
| Phase 4 | 파일럿 실제 ROI 측정 (4기둥 × EARS 연동) |
| Phase 5 | 분기별 ROI 추적, 포트폴리오 최적화 |

**ROI 시뮬레이터 도구:** ICEP 점수 입력 → 예상 ROI 자동 산출 → 제안서에 삽입 가능 (Step 3에서 Excel로 제작 예정)
