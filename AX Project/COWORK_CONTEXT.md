# FLOW~ AX Project — Cowork 컨텍스트 브리핑

> 이 파일은 Claude(Max)와 데미안이 진행한 대화의 전체 맥락을 담고 있습니다.
> Cowork Project에 이 파일을 포함시키면, 새 세션에서도 이전 대화의 맥락을 완전히 이어받을 수 있습니다.

---

## 1. 프로젝트 배경

### 발단
- 국내 대기업(H그룹) 임원으로부터 AX(AI 전환) 관련 문의가 들어옴
- 1차 전화 통화 완료, 후속 미정
- 본격적인 컨설팅 프로젝트로 이어질지는 아직 모르는 상태

### 참고 자료
- 특정인의 블로그/SNS에서 공유된 기업 AX 전략 글을 참고 자료로 분석
- 이 글은 데미안이 작성한 것이 아님 (최초 혼동이 있었으나 명확히 정정됨)
- 참고 글의 핵심 구조: WBC(Work Block Canvas) 9블록 + ICE 매트릭스 + 3 Wave 확산 + 변화관리 4유형
- 참고 글 저자의 고유 자산(WBC 명칭, 9블록 구체 구성)은 가져오지 않음

### 의사결정 과정
1. 참고 글 분석 → "범용 원리"와 "저자 고유 자산" 분리
2. 데미안의 기존 자산(STAR-AR, EARS, ARQ, WHY-WHAT-HOW, HRD 경험, 200+ AI 앱) 정리
3. 데미안 고유의 AX 방법론 개발을 목표로 결정
4. 전체 로드맵 먼저 그리기로 함
5. 타깃: 복수 타깃 (대기업 임원, HRD부서, 중견기업 등 유동적)
6. 6개 Phase 전체 상세 설계 완료

## 2. 참고 글 분석 결과 (가져올 것 / 가져오면 안 되는 것)

### 자유롭게 활용 가능한 범용 원리
- "업무 가시화가 AX의 출발점" — 경영학 기본 원리 (드러커)
- ICE 매트릭스 — Sean Ellis의 그로스해킹 범용 프레임워크
- 변화관리 저항 유형 분류 — Kotter, Prosci ADKAR 등에서 일반화
- 파일럿 → 확산 단계적 접근 — 범용 프로젝트 관리 원칙
- 3 Wave 로드맵 구조 — 일반적 전략 프레임

### 가져오면 안 되는 저자 고유 자산
- "워크블록캔버스(Work Block Canvas, WBC)" 명칭과 9블록 구조
- WBC 9블록의 구체적 구성과 순서
- 해당 글의 특유한 문장/표현

### 참고 글의 한계점 (데미안이 보완해야 할 부분)
- AI 리터러시 교육이 선행 조건으로 빠져 있음
- 업무만 가시화하고 역량/역할 가시화가 없음
- ICE에 사람 준비도 축이 없음
- AI-Native 재설계가 선언적이고 워크숍 방법론이 없음
- 변화관리가 분류만 있고 코칭/퍼실리테이션 실행법이 없음
- 비용/ROI 모델이 없음
- 산업/규모별 차이 미반영
- 데이터 보안/컴플라이언스 얇음
- 중간관리자 전환 전략 없음
- 실패 시나리오 대응 없음
- 노조/직원 대표 소통 전략 없음

## 3. FLOW~ AX 방법론 구조 (완성된 부분)

### 전체 로드맵
Phase 0 → Phase 1 → Phase 2 → Phase 3 → Phase 4 → Phase 5 (피드백 루프)

### Phase별 요약

#### Phase 0: AI 리터러시 선행 교육 (2주)
- 기본반 (Week 1): AI 개념 → 핸즈온 → 미니 프로젝트 → 공유 → 정리
- 실무반 (Week 2): 프롬프트 심화 → 도구 매핑 → 워크플로우 → 팀 협업 → Phase 1 연결
- 성공 기준: 독립 사용 90%+, 긍정 응답 80%+, Phase 1 참여 의향 85%+
- 상세 설계 상태: 뼈대 완성, Day별 상세 퍼실리테이션 가이드 미작성

#### Phase 1: 역할·업무·역량 가시화 (2-3주)
- 고유 도구: RTC 캔버스 (Role-Task-Competency Canvas)
- R영역: 핵심가치(R1) + 역할경계(R2) + 성장벡터(R3)
- T영역: 업무+시간(T1) + 입출력구조(T2) + AI적합도3단계(T3)
- C영역: 현재역량(C1) + AI협업역량(C2) + 전환역량갭(C3)
- 산출물: 조직 업무 지도 + AI 적용 후보군 + 역량 갭 리포트
- 상세 설계 상태: 구조 완성, Notion/앱 템플릿 미제작

#### Phase 2: AI 적용 우선순위 결정 (1-2주)
- 고유 도구: ICEP 매트릭스 (ICE + People Readiness)
- I(Impact) + C(Confidence) + E(Ease) + P(People Readiness)
- 각 축별 평가 항목, 점수 기준(1-10), 판정 기준 완성
- Wave 분류: 8+ 즉시, 6-7 단기, 4-5 중기, 3- 보류
- P축 차별점 예시 완성 (재무팀 사례)
- 상세 설계 상태: 프레임워크 완성, 스코어링 워크숍 퍼실리테이션 가이드 미작성, 스프레드시트 도구 미제작

#### Phase 3: AI-Native 업무 재설계 (2-4주)
- WHY-WHAT-HOW 코칭 프레임 × AX 확장
- WHY 세션(2h): As-Is 시각화 → 데이터 확인 → "백지 설계" 질문
- WHAT 세션(3h): To-Be 설계 → 3단계 분류 → AI 도구 매핑
- HOW 세션(3h): 세팅 → 프롬프트 → 핸드오프 → 검수 → 시연
- 직군별 시나리오 대상: 마케팅, 재무, 인사, 영업, 고객지원, R&D
- 상세 설계 상태: 워크숍 구조 완성, 직군별 As-Is→To-Be 시나리오 미작성, 표준 프롬프트 라이브러리 미작성

#### Phase 4: 파일럿 실행 + 변화관리 (4-6주)
- 파일럿 3단계: 준비(W1) → 실행(W2-4) → 평가(W5-6)
- STAR-AR 변화관리: S(진단) → T(과제) → A(행동) → R(결과) → A(대안) → R(성찰)
- 저항 4유형별 대응: 두려움형/무능형/관성형/불신형
- AX Champion 육성: 3축 선발 + 2주 집중 교육 + 이후 역할
- 실패 시나리오 3경로: 근접→연장, 원인명확→2차, 근본문제→Phase2 복귀
- 상세 설계 상태: 구조 완성, 코칭 스크립트 미작성, Champion 평가표 미제작

#### Phase 5: 측정·확산·거버넌스 (지속)
- EARS + ARQ 성과 측정: E(효과성) A(정렬도) R(적합도) S(지속가능성)
- 거버넌스 4영역: 데이터정책, 품질정책, 도구정책, 책임정책
- 3 Wave 확산: W1(Quick Win) → W2(구조혁신) → W3(전략전환)
- 노조 소통: "업무 고도화" 프레이밍 + 투명성 확보
- 상세 설계 상태: 프레임워크 완성, EARS 대시보드 미제작, 거버넌스 문서 미작성
