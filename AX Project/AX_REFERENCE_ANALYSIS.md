## 참고 글 핵심 구조 요약

### 6개 원칙 + 3 Wave + 목적지

1. 보이지 않는 것은 바꿀 수 없다 → 업무 가시화 (WBC)
2. AI를 어디에 먼저 붙일 것인가 → 우선순위 (ICE 매트릭스)
3. AI를 끼워 넣지 말고 처음부터 다시 설계하라 → AI-Native Design
4. 기술 문제가 아니라 사람 문제다 → 변화관리 (저항 4유형)
5. 파일럿 없이 전사 도입은 없다 → 파일럿 실행
6. 측정하지 않으면 개선할 수 없다 → 성과 측정 + 거버넌스
7. 3단계 웨이브 전략 → Wave 1(즉시) / Wave 2(구조) / Wave 3(전략)
8. AX의 목적지 → 실행자→전략가, 생산자→디렉터

### 저자 고유 도구: WBC (Work Block Canvas) 9블록
1. 역할 정의
2. 핵심 업무 목록
3. 시간 배분 현황
4. 업무 입출력 구조
5. AI 대체 가능성
6. 협업 구조
7. 성과 기준
8. 성장 방향
9. AI 도구 매핑

### 저자의 ICE 매트릭스 적용
- Impact + Confidence + Ease, 3으로 나눔
- 8+ 즉시, 6-7 단기(1-3개월), 4-5 중기, 3- 보류

### 저항 유형 4분류
- 두려움형: "AI가 내 일자리를 빼앗는다"
- 무능형: "나는 AI를 못 쓴다"
- 관성형: "지금도 잘 되는데 왜 바꾸나"
- 불신형: "AI가 틀리면 누가 책임지나"

### 파일럿 성공 기준 (저자 제시)
- 처리 시간 40% 이상 단축
- AI 출력물 재수정률 5% 미만
- 팀원 만족도(사용 의향) 80% 이상
- C영역 집중 시간 3배 이상 증가

### 성과 측정 5차원 (저자 제시)
- 효율성 (처리 시간, 오류율)
- 생산성 (시간당 아웃풋)
- 품질 (재수정율, 고객 만족도)
- 역량 전환 (C영역 집중 시간)
- 비즈니스 성과 (매출, 비용 절감)

## 참고 글의 강점

- 전체 논리 흐름이 명확 (가시화 → 우선순위 → 재설계 → 변화관리 → 파일럿 → 측정 → 확산)
- WBC라는 구체적 도구를 중심에 놓은 것이 설득력 있음
- "잘못된 질문에서 출발한다"는 도입부가 강력함
- "일의 층위가 바뀐다"는 표현이 AX의 본질을 잘 포착
- ICE 점수 기반 의사결정이 회의실 주관적 싸움을 없앤다는 포인트가 좋음

## 참고 글의 한계점 (FLOW~ 방법론이 보완하는 부분)

1. AI 리터러시 교육 선행 단계 없음
   → FLOW~ Phase 0으로 보완

2. 업무만 가시화, 역할/역량 통합 없음
   → FLOW~ RTC 캔버스 (Role + Task + Competency)

3. ICE 3축에 사람 준비도 축 없음
   → FLOW~ ICEP 매트릭스 (P축 추가)

4. AI-Native 재설계가 선언적, 워크숍 방법론 없음
   → FLOW~ WHY-WHAT-HOW 워크숍

5. 변화관리가 분류만, 코칭/퍼실리테이션 없음
   → FLOW~ STAR-AR 코칭 프레임

6. 성과 측정이 일반 지표 나열
   → FLOW~ EARS + ARQ 3중 검증

7. 비용/ROI 모델 없음
   → 미완성 (Cowork에서 작업 필요)

8. 산업/규모별 차이 미반영
   → 미완성 (직군별 시나리오로 일부 보완, 산업별은 추가 필요)

9. 데이터 보안/컴플라이언스 얇음
   → FLOW~ Phase 5 거버넌스 4영역으로 보완

10. 중간관리자 전환 전략 없음
    → 미완성 (별도 설계 필요)

11. 실패 시나리오 없음
    → FLOW~ Phase 4 실패 3경로

12. 노조/직원 대표 소통 없음
    → FLOW~ Phase 5 소통 전략

13. AX Champion 육성 구체적 방법론 없음
    → FLOW~ Phase 4 Champion 육성 프로그램
