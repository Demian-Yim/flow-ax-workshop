# FLOW~ AX Methodology
## 기업 AI 전환(AX) 방법론 — 사람과 일의 흐름을 디자인합니다

**설계자:** 임정훈 (Demian) | AI 코디네이터 | 200+ AI 앱 빌드 · AI 협업 인프라 설계 전문가 · HRD 15년 경력
**브랜드:** FLOW~ (flowdesign.ai.kr)
**최종 업데이트:** 2026-03-21

---

## 방법론 개요

FLOW~ AX Methodology는 HRD 기반의 사람 중심 기업 AI 전환 방법론이다.
기술 컨설턴트가 AI 도구를 도입하고 떠난 자리에서, FLOW~는 사람이 전환을 완수할 때까지 함께한다.

### 핵심 포지셔닝

기존 AX 컨설팅이 "어떤 AI 툴을 쓸 것인가"에서 출발한다면, FLOW~ AX는 "사람이 어떻게 전환할 것인가"에서 출발한다. 업무 가시화뿐 아니라 역할과 역량을 동시에 가시화하고, 기술적 우선순위뿐 아니라 사람의 준비도를 동시에 평가하며, 도입 선언이 아니라 코칭과 퍼실리테이션으로 실행을 완수한다.

### 전체 Phase 구조

```
Phase 0 → Phase 1 → Phase 2 → Phase 3 → Phase 4 → Phase 5
AI 리터러시  역할·업무·역량  우선순위 결정  AI-Native   파일럿 실행   측정·확산
선행 교육    가시화(RTC)    (ICEP)       업무 재설계   + 변화관리    거버넌스
(2주)       (2-3주)       (1-2주)      (2-4주)      (4-6주)     (지속)
```

Phase 5 완료 후 피드백 루프를 통해 Phase 0으로 순환하며 지속적으로 고도화한다.

### 일반 AX 접근 vs FLOW~ AX 차별점

| 구분 | 일반 AX 접근 | FLOW~ AX |
|---|---|---|
| 출발점 | 업무 가시화 | AI 리터러시 교육 선행 (Phase 0) |
| 가시화 도구 | 업무 중심 캔버스 | RTC 캔버스 — 역할+업무+역량 통합 |
| 우선순위 | ICE 3축 | ICEP 4축 — 사람 준비도(P) 추가 |
| 재설계 방법 | 선언적 원칙 제시 | WHY-WHAT-HOW 워크숍 방법론 |
| 변화관리 | 저항 유형 분류 | STAR-AR 코칭 프레임 적용 |
| 성과 측정 | 차원 나열 | EARS + ARQ 3중 검증 체계 |
| 확산 전략 | 3 Wave | 3 Wave + AX Champion + 교육 연계 |
| 실패 대응 | 없음 | 3경로 실패 시나리오 사전 정의 |
| 노조 소통 | 없음 | 프레이밍 전략 + 투명성 확보 |

### 데미안 고유 자산 — 방법론의 기반

1. HRD 15년 현장 경험: 교육 설계, 퍼실리테이션, 코칭, 조직개발 실전
2. 200+ AI 앱 빌드 경험: 이론이 아닌 실전 검증
3. 고유 프레임워크: STAR-AR, EARS 1-Pager, ARQ, WHY-WHAT-HOW
4. 다중 AI 플랫폼 운용 체계: Emil(GPT), Eva(Gemini), Max(Claude), Pistorius(Perplexity)
5. 퍼실리테이션·코칭 전문: 워크숍 설계·실행 역량
6. AI 리터러시 교육 커리큘럼: 기본반·실무반 운영 경험

## Phase 0: AI 리터러시 선행 교육

### 목적

조직 구성원이 AI의 가능성과 한계를 체감하고, 이후 Phase에서 자기 업무에 AI를 매핑할 수 있는 기초 역량을 확보한다.

### 기간

2주 (기본반 1주 + 실무반 1주)

### 왜 Phase 0가 필요한가

일반 AX 접근은 바로 업무 가시화로 들어간다. 그러나 직원들이 AI가 무엇을 할 수 있는지 모르면 이후 모든 단계의 품질이 떨어진다. Phase 1에서 "AI 적합도 분류"를 하려면, 먼저 AI가 뭘 할 수 있는지 알아야 한다. 이것이 FLOW~ AX의 첫 번째 차별점이다.

### Week 1: 기본반

대상: AI를 처음 접하거나 두려움이 있는 구성원
핵심: "AI가 나를 대체하는 것이 아니라, 내 업무를 돕는 도구"라는 인식 전환

- Day 1: AI 개념과 작동원리 — 일상 언어로 설명, 데모 중심
- Day 2: AI 도구 핸즈온 — ChatGPT/Gemini/Claude 직접 사용
- Day 3: 내 업무 AI 미니 실험 — 자기 업무 중 하나를 골라 AI로 시도
- Day 4: 결과 공유 + 피드백 — 팀 내 공유, 퍼실리테이션
- Day 5: AI 가능/한계 정리 — Phase 1 연결 브리핑

### Week 2: 실무반

대상: 기본반 수료자 또는 이미 AI를 일부 활용하는 구성원

- Day 1: 프롬프트 엔지니어링 심화 — 멀티턴 대화 설계, 역할 지정, 구조화된 출력
- Day 2: 업무별 AI 도구 매핑 실습 — 텍스트/이미지/데이터/코드 영역별 최적 도구 탐색
- Day 3: AI 워크플로우 설계 — 입력→AI 처리→인간 검수→출력 파이프라인 설계
- Day 4: 팀 단위 AI 협업 시나리오 설계
- Day 5: Phase 1 업무 가시화 사전 워크숍 — RTC 캔버스 작성법 사전 교육

### 성공 기준

- AI 도구 1개 이상 독립 사용 가능 비율: 90% 이상
- "AI가 내 업무에 도움이 될 수 있다" 긍정 응답: 80% 이상
- Phase 1 업무 가시화 참여 의향: 85% 이상

## Phase 1: 역할·업무·역량 가시화

### 목적

조직 내 모든 구성원의 역할·업무·역량을 표준화된 방식으로 가시화하여, AI 적용의 근거 데이터를 확보한다.

### 기간

2~3주

### FLOW~ 고유 도구 — RTC 캔버스 (Role-Task-Competency Canvas)

#### R 영역 (Role — 나는 누구인가)

- R1. 핵심 가치 정의: 내가 조직에 기여하는 고유 가치는 무엇인가
- R2. 역할 경계: 내 역할의 시작과 끝은 어디인가, 누구와 겹치는가
- R3. 성장 벡터: AI 시대에 내 역할은 어느 방향으로 진화해야 하는가

#### T 영역 (Task — 나는 무엇을 하는가)

- T1. 업무 목록 + 시간 배분: 모든 업무와 실제 투입 시간
- T2. 입출력 구조: 각 업무의 입력물·산출물·의존 관계
- T3. AI 적합도 분류: 완전자동화 / AI보조+인간감독 / 인간전용 (3단계)

#### C 영역 (Competency — 나는 무엇을 할 수 있는가)

- C1. 현재 역량 수준: 현 업무 수행에 필요한 역량과 자기 평가
- C2. AI 협업 역량: AI 도구 활용 수준 (Phase 0 데이터 연동)
- C3. 전환 역량 갭: 새 역할에 필요한 역량과 현재 수준의 차이

#### C 영역이 만드는 결정적 차이

업무만 가시화하면 "이 업무를 AI가 할 수 있다"까지는 판단하지만, "이 사람이 전환할 준비가 됐는가"는 판단하지 못한다. C 영역이 있어야 Phase 4의 변화관리에서 개인별 맞춤 지원이 가능하다.

### 실행 방법

- Week 1: 전 직원 RTC 캔버스 작성 워크숍 (자기 작성→교차 검증→팀장 확인)
- Week 2: 팀 단위 RTC 데이터 취합 → 조직 업무 지도(Work Map) 생성
- Week 3: 업무 지도 기반 AI 적용 후보군 도출

### 산출물

- 조직 업무 지도(Work Map), AI 적용 후보군 리스트, 역량 갭 리포트

## Phase 2: AI 적용 우선순위 결정

### 목적

Phase 1에서 도출된 AI 적용 후보군에 대해, 비즈니스 효과와 사람 준비도를 동시에 고려한 우선순위를 수치로 결정한다.

### 기간

1~2주

### FLOW~ 고유 도구 — ICEP 매트릭스

I (Impact, 영향력, 1-10): 비즈니스 성과 직결 정도
C (Confidence, 신뢰도, 1-10): AI 기술의 정확도와 신뢰성
E (Ease, 용이성, 1-10): 기술적·조직적 도입 난이도
P (People Readiness, 사람 준비도, 1-10): 심리적·역량적 전환 준비도

ICEP = (I + C + E + P) / 4

| ICEP 점수 | 판정 | 배치 |
|---|---|---|
| 8점 이상 | 즉시 실행 | Wave 1 |
| 6~7점 | 단기 실행 | Wave 2 |
| 4~5점 | 중기 과제 | Wave 3 |
| 3점 이하 | 보류/재검토 | - |

### P축이 만드는 차이 — 실제 예시

재무팀 월간 보고서 자동 생성:
- ICE만: I=9, C=8, E=7 → 8.0점 → 즉시 실행
- ICEP: I=9, C=8, E=7, P=3 → 6.75점 → 단기 실행으로 조정 + P점수 선행 조치

### 산출물

- ICEP 점수표, Wave별 분류 리스트, P점수 저조 항목 선행 조치 계획

## Phase 3: AI-Native 업무 재설계

### 목적

ICEP 기반 선정 업무에 대해, AI 전제 하에 업무 프로세스를 처음부터 새로 설계한다.

### 기간

2~4주

### FLOW~ 고유 방법론 — WHY-WHAT-HOW 워크숍

#### WHY 세션 (2시간): 왜 재설계하는가

- As-Is 프로세스 시각화, 시간·오류 데이터 확인
- 핵심 질문: "이 업무를 오늘 처음 만든다면, AI가 있는 세상에서 어떻게?"

#### WHAT 세션 (3시간): 무엇을 바꾸는가

- To-Be 프로세스 설계
- 3단계 분류: AI 완전자동화 / AI보조+인간감독 / 인간전용
- AI 도구 후보 매핑 (다중 AI 플랫폼 운용 경험 직접 활용)

#### HOW 세션 (3시간): 어떻게 실행하는가

- AI 세팅, 프롬프트 설계, 인간-AI 핸드오프 정의
- 품질 검수 기준, 예외 처리 프로세스
- 실제 AI 도구로 To-Be 프로세스 실시간 시연

### 산출물

- 직군별 As-Is→To-Be 전환 시나리오, 3단계 분류표, AI 도구 매핑표, 프롬프트 설계서

## Phase 4: 파일럿 실행 + 변화관리

### 목적

Wave 1 대상 파일럿 실행 + HRD 코칭 기반 저항관리 + AX Champion 육성

### 기간

4~6주

### 파일럿 3단계

1단계 준비(W1): 팀 선정, 성공 기준 합의, 베이스라인 수집
- 처리 시간 40%+ 단축, 재수정률 5% 미만, 만족도 80%+, C영역 시간 3배+

2단계 실행(W2-4): To-Be 적용, 주 1회 체크인, 현장 코칭

3단계 평가(W5-6): 성과 측정, 회고 워크숍, 롤아웃 판단, Champion 선발

### 변화관리 — STAR-AR 프레임

- S(Situation): 저항 유형 진단 (두려움/무능/관성/불신)
- T(Task): 유형별 전환 과제 정의
- A(Action): 1:1 코칭, 팀 퍼실리테이션, 피어 러닝
- R(Result): AI 사용 빈도·만족도·전환율 측정
- A(Alternative): 플랜 B 대안 제시
- R(Reflection): 과정 성찰 및 학습

### AX Champion 육성

선발: AI 활용 + 동료 영향력 + 실험 의지 3축
교육: 2주 집중 (AI 심화 + 퍼실리테이션 + 변화관리)
운영: 팀별 AI 길잡이 + 월 1회 네트워크 모임

### 실패 시나리오 3경로

1. 기준 근접(90%+): 2주 연장 재평가
2. 원인 명확: 원인 제거 후 2차 파일럿
3. 근본 문제: Phase 2 복귀 ICEP 재평가

---

## Phase 5: 측정·확산·거버넌스

### 성과 측정 — EARS + ARQ

- E(Effectiveness): 처리 시간·오류율·산출물·비용
- A(Alignment): 전략 목표 기여도·부서 간 일관성
- R(Relevance): 만족도·재수정률·지속 사용률(30/60/90일)
- S(Sustainability): Champion 지속률·자발적 확산·역량 전환 진행도
- ARQ 3중 검증: 각 차원의 정확성·적합성·품질 검증

### 3 Wave 확산

- Wave 1 (1-4주, ICEP 8+): Quick Win — 즉시 성과, 주 100h+ 절감
- Wave 2 (2-3개월, ICEP 6-7): 구조 혁신 — 프로세스 재설계, 리드타임 60%↓
- Wave 3 (4-6개월, ICEP 4-5): 전략 전환 — 고가치 판단, 신규 비즈니스

### 거버넌스 4영역

1. 데이터 정책: 등급별 AI 입력 기준
2. 품질 정책: 검수·승인 프로세스
3. 도구 정책: 승인 도구 목록·도입 심사
4. 책임 정책: 법적·윤리적 책임 구조

### 노조·직원 대표 소통

- "일자리 대체"가 아닌 "업무 고도화" 프레이밍
- 파일럿 결과 투명 공유, 역할 전환 로드맵 사전 제시
- 노조 대표 파일럿 옵서버 초대
