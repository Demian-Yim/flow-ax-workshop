## 목적
WS06 파일럿을 실행하기 전, NIST AI RMF의 4대 기능(Govern·Map·Measure·Manage)에 따라 책임성·안전성·투명성 체크포인트를 점검한다.

## 작성 가이드
1. 각 항목에 [x]로 체크하고, 해당없음은 "N/A"로 표기한다.
2. 미체크 항목에는 보완 계획과 담당자를 적는다.
3. CRITICAL 항목 중 하나라도 미체크이면 파일럿을 착수하지 않는다.
4. 4주마다 재점검하고, 변경 이력을 하단에 기록한다.
5. 법무/보안 담당자 서명 후 실행한다.

## GOVERN — 거버넌스 (조직·정책)
- [ ] **(C)** 사용 AI 모델·벤더의 이용약관·데이터 처리 정책을 확인했다
- [ ] **(C)** 사내 AI 사용 정책 / 가이드라인에 부합한다
- [ ] 담당자(Owner)와 승인자(Approver)가 지정되어 있다
- [ ] 이해관계자(법무·보안·HR·현업)에 공지되었다

## MAP — 맥락과 리스크 식별
- [ ] **(C)** 처리 데이터에 개인정보·기밀·영업비밀 포함 여부를 식별했다
- [ ] 이 유즈케이스의 직접/간접 사용자를 명시했다
- [ ] 오작동 시 최악의 시나리오(Worst case)를 1개 이상 적었다
- [ ] 편향(Bias) 발생 가능 지점을 식별했다

## MEASURE — 측정과 평가
- [ ] WS05 EARS 지표 중 품질·안전 지표가 포함되어 있다
- [ ] 환각(hallucination) / 오답률을 측정하는 방법이 있다
- [ ] 인간 검수 비율과 샘플링 기준이 정의되어 있다
- [ ] 사용자 피드백 수집 경로가 있다

## MANAGE — 관리와 대응
- [ ] **(C)** 롤백 절차(WS06 참조)가 문서화되어 있다
- [ ] 사고 발생 시 보고 경로(누구에게, 몇 시간 내)가 정해져 있다
- [ ] 로그/프롬프트/출력 보관 기간과 삭제 정책이 있다
- [ ] 모델/프롬프트 변경 이력을 추적한다
