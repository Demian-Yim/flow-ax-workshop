# Phase -1 · 준비도 진단

## M01 · 데이터 준비도 진단

- **모듈 ID:** M01
- **제목:** 데이터 준비도 진단 (Data Readiness Assessment)
- **Phase:** -1 (준비도 진단)
- **3축:** S (컨설팅 주도) + 보조 F

### 학습목표 (Bloom)
1. 조직의 데이터 자산 현황을 **진단한다 (Analyze)**.
2. AX 착수 전 필수 데이터 요건 5가지를 **설명한다 (Understand)**.
3. 준비도 점수를 기반으로 개선 우선순위를 **판단한다 (Evaluate)**.

### 대상 · 인원 · 시간
- **대상:** 고객사 IT/DX 실무자 + 데이터 오너 (부서장급)
- **인원:** 6~12명
- **시간:** 4시간 (반나절 집중)

### 준비물
- 데이터 자산 인벤토리 템플릿 (엑셀)
- 접근권한/보안 정책 사본
- 진단 설문지 (40문항)
- 노트북 1인 1대, 빔프로젝터, 포스트잇

### 분단위 타임라인
| 분 | 내용 |
|---|---|
| 00~10 | 오리엔테이션 · AX와 데이터의 관계 |
| 10~40 | 강의: 데이터 준비도 5요건 (존재/접근/품질/정합/거버넌스) |
| 40~80 | 개인 진단 설문 작성 (40문항) |
| 80~100 | 휴식 |
| 100~150 | 팀별 데이터 인벤토리 작성 (3~5개 주요 데이터셋) |
| 150~200 | 팀별 발표 & 상호 질의 |
| 200~230 | 진단 결과 스코어링 · 히트맵 작성 |
| 230~240 | 다음 단계 안내 · 마무리 |

### 핵심내용
- 데이터 5요건: 존재성 · 접근성 · 품질 · 정합성 · 거버넌스
- "데이터가 없으면 AX도 없다" — 착수 전 필수 전제
- 준비도 점수 = (가중합 / 만점) × 100, 60점 미만은 선행작업 필요
- 흔한 함정: 엑셀 곳곳 흩어짐, 오너 불명, 민감정보 미분류

### 사용 워크시트
- WS-01 데이터 인벤토리 시트
- WS-02 준비도 40문항 진단표
- WS-03 히트맵 캔버스

### 강사 노트
- **주의:** IT팀과 현업팀의 데이터 이해도 차이가 크다. 용어부터 합의할 것.
- **흔한 실수:** 점수에만 집착하고 맥락을 놓침 → "왜 낮은가"를 반드시 토론
- **팁:** 민감정보 관련 답변은 익명 처리 보장 약속 먼저

### 평가 방법
- 진단 설문 완료율 100%
- 인벤토리 정확도 (컨설턴트 리뷰)
- 개선 과제 3개 이상 도출 여부

### 산출물
- 데이터 준비도 진단 보고서 (10~15p)
- 히트맵 1장
- 개선 우선순위 백로그

## M02 · AI 성숙도 베이스라인

- **모듈 ID:** M02
- **제목:** AI 성숙도 베이스라인 측정
- **Phase:** -1
- **3축:** S + E

### 학습목표
1. 조직의 AI 성숙도를 5단계 모델로 **평가한다 (Evaluate)**.
2. 부서별/직무별 격차를 **비교한다 (Analyze)**.
3. 1년 후 목표 성숙도를 **설정한다 (Create)**.

### 대상 · 인원 · 시간
- **대상:** 임원 + 부서장 + HR/DX
- **인원:** 8~15명
- **시간:** 3시간

### 준비물
- 성숙도 설문 온라인 폼 (사전 배포)
- 부서별 집계표
- 벤치마크 자료 (업종별)

### 분단위 타임라인
| 분 | 내용 |
|---|---|
| 00~15 | 사전 설문 결과 요약 공유 |
| 15~45 | 5단계 성숙도 모델 강의 (미인식→실험→부분도입→전사확산→AI네이티브) |
| 45~75 | 부서별 현재 위치 표시 (스티커 투표) |
| 75~90 | 휴식 |
| 90~130 | 갭 분석 워크숍 (As-Is vs 1년후 To-Be) |
| 130~165 | 우선 투자 영역 3개 도출 |
| 165~180 | 합의 · 책임자 지정 · 마무리 |

### 핵심내용
- 5단계 성숙도 모델 (Lv0 미인식 → Lv4 AI 네이티브)
- 축: 인식/스킬/도구/프로세스/거버넌스 5축 평가
- 부서 간 격차가 전사 확산의 병목
- 목표는 "1단계 상승"이 현실적

### 사용 워크시트
- WS-04 성숙도 5축 레이더 차트
- WS-05 부서별 위치 매핑 보드
- WS-06 갭 분석 시트

### 강사 노트
- **주의:** 점수 낮은 부서가 창피해하지 않도록 익명 집계
- **흔한 실수:** 목표를 "Lv4"로 과욕 → 현실성 없어 실행력 하락
- **팁:** 임원 직접 참여 시 솔직도 급상승. 경영진 참여 필수.

### 평가 방법
- 설문 응답률 80%+
- 갭 분석표 완성도
- 목표 수준 합의 여부

### 산출물
- AI 성숙도 베이스라인 리포트
- 부서별 레이더 차트
- 1년 목표 합의문

# Phase 0 · 기초 리터러시

## M03 · AI 리터러시 기본반

- **모듈 ID:** M03
- **제목:** AI 리터러시 기본반 (전원 공통)
- **Phase:** 0
- **3축:** E (교육)

### 학습목표
1. 생성형 AI의 작동 원리를 비개발자 언어로 **설명한다 (Understand)**.
2. 업무에서 AI를 쓸 수 있는 상황/없는 상황을 **구분한다 (Analyze)**.
3. 기본 프롬프트 5종을 **작성한다 (Apply)**.

### 대상 · 인원 · 시간
- **대상:** 전 직원 (직급/직무 무관)
- **인원:** 20~40명 (대집단)
- **시간:** 4시간

### 준비물
- ChatGPT/Claude/Gemini 계정 (사전 발급)
- 실습용 업무 샘플 3종
- 프롬프트 치트시트

### 분단위 타임라인
| 분 | 내용 |
|---|---|
| 00~20 | 오프닝 · 왜 지금 AI인가 |
| 20~60 | 생성형 AI 작동원리 (토큰/문맥/환각) |
| 60~90 | 데모: 같은 질문 3가지 도구 비교 |
| 90~105 | 휴식 |
| 105~150 | 실습 1: 기본 프롬프트 5종 작성 |
| 150~195 | 실습 2: 본인 업무에 적용 |
| 195~225 | 공유 & 피드백 |
| 225~240 | Q&A · 마무리 |

### 핵심내용
- AI는 **검색엔진이 아니라 생성기** — 답을 만들어낸다
- 토큰/문맥창/환각(Hallucination) 개념
- 프롬프트 = 역할 + 맥락 + 과업 + 형식
- "안 쓰는 게 손해"가 되는 업무 5가지

### 사용 워크시트
- WS-07 프롬프트 기본 5종 템플릿
- WS-08 AI 적합/부적합 판별 체크리스트

### 강사 노트
- **주의:** 용어 폭탄 금지. "LLM" 대신 "AI 글쓰기 도구"
- **흔한 실수:** 데모만 화려하고 실습 시간 부족 → 반드시 실습 60분 이상 확보
- **팁:** 개인 업무 예시를 먼저 듣고 그 예시로 데모

### 평가 방법
- 실습 5종 제출
- 소감문 (AI에 대한 이전/이후 생각)
- 간단 퀴즈 10문항

### 산출물
- 개인 프롬프트 노트
- 수료 뱃지 (Lv1)

# Phase 1 · 현재 파악

## M05 · RTC 캔버스 워크숍

- **모듈 ID:** M05
- **제목:** RTC 캔버스 워크숍 (Role-Task-Context)
- **Phase:** 1
- **3축:** F (퍼실)

### 학습목표
1. 팀의 역할/과업/맥락을 **구조화한다 (Organize)**.
2. 팀원 간 업무 이해도 격차를 **드러낸다 (Analyze)**.
3. AX 적용 후보 영역을 **선정한다 (Evaluate)**.

### 대상 · 인원 · 시간
- **대상:** 단일 팀 전원 (팀장 포함)
- **인원:** 5~10명
- **시간:** 3시간

### 준비물
- RTC 캔버스 대형 인쇄 (A1)
- 포스트잇 4색, 마커
- 타이머

### 분단위 타임라인
| 분 | 내용 |
|---|---|
| 00~10 | 워크숍 목적 공유 |
| 10~20 | RTC 캔버스 소개 |
| 20~50 | 개인 작성 (역할/과업/맥락) |
| 50~80 | 팀 통합 (캔버스 위에 포스트잇) |
| 80~90 | 휴식 |
| 90~120 | 중복/공백 영역 식별 |
| 120~150 | AX 후보 영역 투표 (점 스티커) |
| 150~180 | 합의 · 다음 단계 |

### 핵심내용
- R(Role): 누가 책임지는가
- T(Task): 무엇을 하는가 (동사 단위)
- C(Context): 어떤 조건/제약/산출물
- 중복과 공백이 드러나는 순간이 핵심

### 사용 워크시트
- WS-12 RTC 개인 작성지
