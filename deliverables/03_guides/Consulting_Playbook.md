## 0. 초심자가 가장 많이 빠지는 함정 TOP 10

1. NDA 서명 전에 고객사 내부 문서·데이터 먼저 수령 → **법적 리스크 + 보험 미적용**
2. SOW에 "산출물: AI 컨설팅 보고서" 식 모호한 기재 → **완료 판정 불가 → 잔금 미수**
3. 견적에 VAT 10% 별도 표기 누락 → **계약금의 9.09%를 자비로 부담**
4. 카톡·개인 Gmail로 산출물 전달 → **보안 사고 + 증빙 불가**
5. ChatGPT Free/Plus 플랜으로 고객 데이터 입력 → **학습 데이터 반영 + 유출**
6. API 키를 데미안 개인 명의로 발급해 고객사에 빌려줌 → **과금 리스크 + 위탁 위반**
7. "추가 요청 한두 개는 그냥 해드릴게요" → **스코프 크리프로 MD 2배**
8. 원천세 3.3% 공제를 모른 채 세금계산서 발행 → **세무 이중 처리**
9. 사고 발생 후 72시간 경과 → **개인정보보호법 과태료 최대 3%**
10. 로컬 디스크 암호화 미설정 노트북을 고객사 미팅에 반입 → **분실 시 기밀 유출**

## 1. 계약 가이드

### 1.1 수주 파이프라인 (6단계)

```
[리드] → [1차 미팅] → [제안서] → [NDA] → [SOW] → [본계약] → [Kickoff]
```

각 단계별 표준 소요: 리드 접수 후 Kickoff까지 **평균 3~6주**.

#### 1.1.1 리드 단계 체크리스트

- [ ] 리드 출처 기록 (소개/세미나/홈페이지/콜드) — 전환율 집계용
- [ ] 담당자 직책·의사결정 권한 확인 (결정권자 / 실무자 / 게이트키퍼)
- [ ] 예산 규모 감지 (직접 묻지 말 것, "유사 프로젝트 5천~1억 범위" 식으로 반응 관찰)
- [ ] 리드 → 고객사 CRM (Notion DB) 등록
- **반드시 할 것:** 첫 연락 후 24시간 내 1차 회신
- **절대 하지 말 것:** 리드 단계에서 상세 제안 수치·방법론 노출

#### 1.1.2 1차 미팅 (디스커버리)

- [ ] 미팅 전: 고객사 공개정보 조사 (IR, 보도자료, 직원수, 매출, 경쟁사) 30분 이상
- [ ] 미팅 어젠다 이메일 선송부 (회의 1영업일 전)
- [ ] **질문 리스트 5종 준비**
  1. 현재 AI/AX 관련 고민 1순위는?
  2. 이미 시도한 것과 결과는?
  3. 성공의 정의(KPI)는 무엇인가?
  4. 프로젝트 일정과 예산 범위는?
  5. 의사결정 프로세스와 최종 승인자는 누구인가?
- [ ] **절대 하지 말 것:** 첫 미팅에서 "얼마면 됩니다" 즉답. 항상 "검토 후 2영업일 내 제안" 응대.
- [ ] 미팅 후 4시간 내 회의록 + Thank-you 메일 발송

#### 1.1.3 제안서 (Proposal)

- [ ] 제안서 필수 구성 10항목
  1. 표지 (프로젝트명, 고객사명, 제안일, 유효기간 30일)
  2. Executive Summary (1페이지)
  3. 현황 진단 (고객의 문제를 고객의 언어로)
  4. 제안 솔루션 (5패키지 중 적합한 것 매핑)
  5. 기대 효과 (정량 + 정성)
  6. 수행 방법론 (Phase / 산출물 / 마일스톤)
  7. 일정표 (간트차트)
  8. 투입 인력 / 역할
  9. 투자 금액 (VAT 별도 명시)
  10. 레퍼런스 / 회사 소개
- [ ] 제안서는 **PDF로만** 배포. Word 원본 배포 금지 (수정·재활용 방지)
- [ ] 제안서 하단 고정 문구: *"본 제안서는 고객사 내부 검토 목적으로만 사용되며, 제3자 공개를 금합니다. 유효기간: 발송일로부터 30일."*

#### 1.1.4 NDA (비밀유지계약)

- [ ] **원칙: NDA 서명본 수령 전에는 고객사 내부 문서·데이터·로그를 단 한 건도 수령하지 않는다.**
- [ ] NDA는 **상호(Mutual) NDA** 체결 (데미안 측 산출물·방법론도 보호)
- [ ] Kickoff 3영업일 전 NDA 서명본 수령 완료
- [ ] NDA 원본은 법률 자문 검토 최소 1회 완료된 템플릿만 사용

##### NDA 표준 조항 (7개)

| 조항 | 내용 | 권장 기준 |
|---|---|---|
| 정의 | 비밀정보의 범위 (구두, 문서, 전자, 시각) | 포괄 정의 |
| 비밀유지 의무 | 제3자 제공·공표 금지 | 상호 의무 |
| 예외사항 | 공지된 정보 / 독자 개발 / 법원 명령 / 사전 동의 | 4가지 한정 |
| 유효기간 | 계약 종료 후 존속기간 | **2~3년** |
| 파기 의무 | 종료 시 반환·파기·증빙서 제출 | 30일 이내 |
| 손해배상 | 위반 시 실손해 + 법정비용 | 상한 없음 |
| 관할법원 | 분쟁 시 관할 | 서울중앙지방법원 |

- **절대 하지 말 것:** 고객사가 제시한 일방(One-way) NDA를 검토 없이 서명
- **반드시 할 것:** NDA 서명 전 자사 변호사(또는 법률 자문) 검토 최소 1회

#### 1.1.5 SOW (Statement of Work, 작업기술서)

SOW는 본계약의 부속문서이자 **분쟁 발생 시 1순위 판단 기준**이다. 모호하면 무조건 데미안이 진다.

##### SOW 필수 10항목

1. **프로젝트명 및 배경**
2. **목표 (Objectives)** — 측정 가능한 3~5개
3. **범위 (Scope In / Scope Out)** — 포함/제외를 **양쪽 모두** 명시
4. **산출물 정의 (Deliverables)** — 포맷, 수량, 품질 기준
5. **완료기준 (Acceptance Criteria)** — 아래 1.1.6 참조
6. **일정 (Schedule & Milestones)**
7. **투입 인력 (Resources)** — 시니어/주니어 MD 분배
8. **고객사 의무 (Client Responsibilities)** — 자료 제공, 담당자 지정, 피드백 기한
9. **변경관리 (Change Request) 프로세스** — 아래 참조
10. **가정 및 리스크 (Assumptions & Risks)**

##### 변경관리(CR) 프로세스 표준

```
고객 요청 → CR 양식 작성 (24h 내) → 영향도 분석 (MD·일정·비용) → 견적 송부 → 서면 승인 → 반영
```

- CR 양식은 **반드시 이메일 또는 PDF 서면**으로. 카톡·구두 승인 무효.
- **절대 하지 말 것:** "이 정도는 서비스로 해드릴게요" — 첫 CR부터 과금 원칙을 명확히 해야 뒤가 편하다.

##### 지연 대응 조항

- 고객 귀책 지연 (자료 미제공, 피드백 지연): **지연일수 × 일 MD 단가 × 0.5 = 대기 비용**으로 청구
- 데미안 귀책 지연: 지체상금은 **총 계약금액의 0.1%/일, 최대 10%** 상한 (관행)
- 불가항력(천재지변, 감염병, 고객사 M&A 등): 양자 합의 일정 재조정

#### 1.1.6 산출물 완료기준 정의법 (모호성 제거)

**나쁜 예:** "AI 활용 교육 자료 1식"
**좋은 예:**
- 포맷: PPTX (16:9, 마스터 슬라이드 FLOW~ 브랜드)
- 분량: 120~150 슬라이드
- 내용: 모듈 1~5, 각 모듈 실습 20분 포함
- 언어: 한국어
- 납품: PPTX 원본 + PDF + 실습파일 zip
- **완료 판정:** 고객사 담당자의 **서면(이메일) 확인** 또는 수령 후 **5영업일 이내 이의 미제기 시 자동 완료**

모든 산출물에 위 6요소 (포맷 / 분량 / 내용 / 언어 / 납품물 / 완료 판정)를 명시한다.

#### 1.1.7 본계약 핵심 조항 체크리스트

| 조항 | 데미안 입장 권장 문구 |
|---|---|
| 지식재산권 | "본 용역의 산출물 중 방법론, 프롬프트 라이브러리, 템플릿, 범용 프레임워크는 용역 제공자(FLOW~)에 귀속된다. 고객 고유 데이터 기반 결과물은 고객에 귀속된다." |
| 사용권 | "고객은 산출물을 자사 내부 교육·업무 목적에 한해 영구·비독점 사용할 수 있다. 제3자 공여·재판매·외부 교육 상품화는 별도 합의." |
| 책임 한계 | "본 계약상 용역 제공자의 손해배상 책임 상한은 **본 계약 총 금액**을 초과하지 아니한다." — 반드시 명시 |
| 보증 제외 | "AI 모델 출력 결과의 사실 정확성, 완전성, 최신성에 대해 보증하지 아니하며, 최종 의사결정 책임은 고객에게 있다." |
| 간접손해 제외 | "일실이익, 기회비용, 영업권 손상 등 간접·결과적 손해는 배상 범위에서 제외한다." |
| 분쟁 해결 | "1차 협의 → 2차 대한상사중재원 중재 → 중재 불성립 시 서울중앙지방법원" |
| 계약해지 | "상대방 중대한 의무 위반 시 14일 서면 시정요구 후 미시정 시 해지 가능" |
| 존속조항 | "비밀유지, 지식재산권, 책임제한은 계약 종료 후에도 존속한다." |

- **반드시 할 것:** 책임 한계 조항. 이게 없으면 수천만원 계약으로 수억 손해배상 리스크.
- **절대 하지 말 것:** 고객사 초안에 "최선을 다한다" / "업계 최고 수준 보장" 같은 모호한 보증 문구 방치.

#### 1.1.8 범위 외 요청 대응 단가표 (2026)

| 구분 | 단위 | 단가(VAT 별도) |
|---|---|---|
| 추가 워크숍 1회 (3h) | 회 | 150만원 |
| 추가 컨설팅 미팅 (1h) | 회 | 30만원 |
| 산출물 추가 수정 (범위 외) | 시간 | 15만원/h |
| 긴급 대응 (48h 내) | 할증 | 기본단가의 **1.5배** |
| 출장 (수도권 외) | 건 | 실비 + 20만원 |
| 별도 보고서 작성 | 건 | 300만원~ |

- 모든 범위 외 요청은 **CR 절차**를 거쳐야 한다. 단가표가 있어도 CR 승인 없이 실행하면 청구 불가.

#### 1.1.9 계약 종료 후 유지보수 범위

계약 종료 후 **30일간 무상 Q&A** (이메일 기준, 회당 30분 상당 답변)만 제공. 이후:

- **월 유지보수 계약**: 월 200만원~ (월 4h 컨설팅 + 자료 업데이트)
- **SOS 단발 호출**: 시간당 20만원 (최소 2시간)
- **절대 하지 말 것:** "언제든지 연락주세요" 구두 약속. 반드시 서면으로 범위 제한.

## 2. 재무 가이드

### 2.1 견적 산정 공식

```
견적(공급가액) = (Σ 역할별 MD × 역할별 단가) + 리스크 버퍼(15%)
총 청구액       = 공급가액 × 1.10 (VAT 10%)
```

**예시:** 시니어 10MD + 주니어 15MD 프로젝트
- 인건비: (10 × 100만) + (15 × 60만) = 1,900만
- 리스크 버퍼 15%: 285만
- 공급가액: 2,185만
- VAT 10%: 218.5만
- **총 청구액: 2,403.5만원**

### 2.2 단가 기준 (2026 기준, FLOW~ 내부)

| 등급 | 일 단가(MD) | 설명 |
|---|---|---|
| Principal (데미안) | 120만원 | 기획·설계·최종 검수·키 미팅 |
| Senior | 100만원 | 메인 실행·워크숍 리드 |
| Junior | 60만원 | 자료 작성·리서치·운영 지원 |
| 외부 전문가 | 150~200만원 | 별도 프로젝트별 협의 |

- 1 MD = 1인 × 8시간
- 단가는 **연 1회(1월) 재조정**. 물가상승률 + 포트폴리오 축적도 반영.

### 2.3 5패키지별 표준 견적 범위 (VAT 별도)

| 패키지 | 기간 | 투입 MD | 표준 범위 |
|---|---|---|---|
| P1. AX 인식 워크숍 (1일) | 1주 | 5~8 MD | 500만 ~ 900만 |
| P2. AX 부서 도입 (4주) | 1개월 | 15~25 MD | 1,800만 ~ 3,000만 |
| P3. 부서 AX 파일럿 (8주) | 2개월 | 30~50 MD | 3,500만 ~ 6,000만 |
| P4. 전사 AX 로드맵 (12주) | 3개월 | 50~80 MD | 6,000만 ~ 1억 |
| P5. 전사 실행 파트너 (6개월+) | 6개월+ | 100+ MD | 1.2억 ~ 3억 |

### 2.4 청구 일정 (권장안)

| 시점 | 비율 | 조건 |
|---|---|---|
| Kickoff 시점 (T-0) | **30%** | 계약 체결 후 7영업일 내 입금 |
| 중간 마일스톤 (중간 보고) | **40%** | 중간 산출물 승인 후 7영업일 내 |
| 최종 납품 (완료) | **30%** | 최종 완료 승인 후 14영업일 내 |

- **소규모 계약(2,000만 미만):** 50% / 50% (Kickoff / 완료)
- **장기 계약(6개월+):** 10% 착수금 + 월 균등 청구

### 2.5 VAT 10% 처리

- **데미안 사업자 유형 = 과세사업자 (일반과세)** 가정 (AX 컨설팅은 면세업종 아님)
- 모든 견적·계약서에 **"VAT 10% 별도"** 명시 (이것 놓치면 실수령이 9.09% 깎인다)
- 세금계산서 발행 시 공급가액과 세액 분리 표기
- **절대 하지 말 것:** "부가세 포함 2천만원" 식 모호한 표기

### 2.6 원천세 3.3% vs 부가세 10% 구분

| 구분 | 원천세 3.3% | 부가세 10% |
|---|---|---|
| 대상 | **사업소득**(프리랜서 개인, 간이과세자 일부) | 과세사업자 용역 |
| 공제자 | 지급자(고객사)가 원천징수 후 송금 | 공급자(데미안)가 수령 후 분기 신고·납부 |
| 세금계산서 | 발행 **안 함** (원천징수영수증) | **반드시 발행** |
| 데미안 경우 | **해당 없음** (일반 과세사업자 기준) | **해당** |

- **핵심:** 데미안이 일반 과세사업자 법인/개인사업자로 세금계산서 발행 체계면 원천세 3.3% 공제는 **없어야 한다**. 고객사가 원천징수하려 하면 "저희는 과세사업자라 세금계산서 발행합니다" 라고 정정.
- 만약 고객사 규정상 원천세 공제가 필수라면 → 그 고객은 데미안을 "프리랜서 사업소득자"로 처리 중 → 세무사 상담 필수.

### 2.7 세금계산서 발행 타이밍

| 시점 | 발행일 | 근거 |
|---|---|---|
| Kickoff 청구 | 입금 요청일 당일 또는 입금일 | 공급시기 = 대가를 받기로 한 때 |
| 중간/완료 청구 | 마일스톤 승인일로부터 **10일 이내** | 부가세법 시행령 |
| 월 정기 청구 | 매월 말일자 | 월 단위 공급 |

- 홈택스 전자세금계산서 발행 **의무** (개인사업자 연매출 8천만 이상, 법인 전부)
- 발행 지연 시 가산세 1% (공급가액 기준)

### 2.8 지연 이자 / 대금 미납 대응

- 계약서에 **연체이자 연 6%** 조항 포함 (상법상 상사법정이율)
- 입금 기한 초과 시 대응 순서:
  1. D+1: 담당자 리마인드 이메일
  2. D+7: 공식 독촉장 (이메일 + 등기)
  3. D+14: 내용증명 발송
  4. D+30: 지급명령 신청 (법원, 소액은 전자소송 가능)
  5. D+60: 민사소송 (변호사 상담)
- **반드시 할 것:** 미납 발생 즉시 추가 작업 중단. "일은 계속 하면서 독촉"은 최악.

### 2.9 개인사업자 vs 법인 세무 차이

| 항목 | 개인사업자 | 법인 |
|---|---|---|
| 세율 | 종합소득세 6~45% 누진 | 법인세 9~24% |
| 설립 비용 | 무료 | 50~100만원 |
| 신고 주기 | 연 1회(5월) | 연 1회(3월) + 중간예납 |
| 대표 보수 | 전액 본인 | 근로소득 처리 (4대보험) |
| 대외 신뢰도 | 낮음 | 높음 (대기업 계약 유리) |
| 권장 전환 시점 | 연 매출 **7천만 초과 지속** 시 법인 전환 검토 |

- **2026년 기준 FLOW~ 초기:** 개인사업자(일반과세)로 시작 → 연매출 1억 근접 시 세무사와 법인 전환 상담

### 3.1 고객사 데이터 분류 3단계

| Level | 분류 | 예시 | 취급 원칙 |
|---|---|---|---|
| **L1 공개** | 대외 공개정보 | 홈페이지, IR, 보도자료 | 자유 사용 |
| **L2 내부** | 직원 대상 문서 | 조직도, 내부 공지, 일반 업무 자료 | NDA 후 취급, 고객사 전용 폴더 |
| **L3 기밀** | 비공개·민감 | 재무, 고객DB, 계약서, 개인정보, 전략 | NDA + 별도 확인서, 암호화 필수, 외부 AI 입력 금지 |

### 3.7 Shadow AI 탐지·양성화

고객사 내부에 승인 없이 쓰이는 AI를 파악하는 절차:

1. **설문:** 전 직원 익명 설문 ("어떤 AI 도구를 업무에 쓰고 계신가요?")
2. **로그 분석:** 방화벽/프록시 로그에서 AI 서비스 도메인 접속 추출
3. **인터뷰:** 부서장 5~10명 1:1
4. **리스크 평가:** 각 도구의 데이터 노출 범위·빈도 스코어링
5. **양성화 정책:** 승인 목록(Whitelist) 발표 + 교육 + 모니터링

### 4.3 API 키 관리 원칙

- **원칙 1: 고객사 프로젝트에 사용되는 API 키는 고객사 명의로 발급받아 사용한다.**
- **원칙 2: 데미안 개인 계정의 API 키를 고객사 작업에 쓰지 않는다.** (과금·감사·위탁 위반)
- **원칙 3: API 키는 절대 코드에 하드코딩하지 않는다.** `.env` + `.gitignore` 필수
- **원칙 4: API 키 로테이션 주기 — 3개월마다 1회, 프로젝트 종료 시 즉시 폐기**
- **원칙 5: 위탁 금지 — 고객사가 API 키를 데미안에게 "알아서 써주세요" 하고 넘기면 반드시 위임장·위탁계약서 작성**
- 키 저장: 1Password / Bitwarden / 클라우드 Secret Manager 사용. 평문 메모 금지.

### 4.4 프로젝트 작업 환경 표준

#### 4.4.1 폴더 구조

```
D:\00 Antigravity\ACTIVE\{프로젝트코드}_{고객사명}\
  ├── 01_contract/       계약서, NDA, SOW, 청구서
  ├── 02_input/          고객 제공 자료 (원본 불변)
  ├── 03_work/           작업 중간 산출물
  ├── 04_deliverables/   최종 납품물 (버전 관리)
  ├── 05_logs/           AI 프롬프트 로그
  ├── 06_meetings/       회의록
  └── README.md          프로젝트 개요 + 연락처 + 링크
```

#### 4.4.2 파일 네이밍 규칙

```
YYYYMMDD_{프로젝트코드}_{문서종류}_{버전}_{작성자이니셜}.{ext}
예: 20260409_FAXW01_SOW_v1.2_DM.pdf
```

- 공백·한글 파일명 가능하나 버전·날짜 필수
- 최종본은 `_FINAL`, 제출본은 `_SUBMIT` 접미사

#### 4.4.3 버전 관리

- 문서: v0.1 (초안) → v0.9 (검토) → v1.0 (납품) → v1.1 (수정)
- 코드: Git 필수. 고객사 프로젝트는 **Private** 리포지토리.
- 중간 백업: Git + 클라우드 이중화

### 4.5 산출물 전달 표준

| 산출물 유형 | 포맷 | 전달 채널 | 보호 |
|---|---|---|---|
| 보고서 | PDF (편집본은 Word/Notion) | 고객사 공식 이메일 or 공유드라이브 | PDF 암호 or 링크 만료 |

- **절대 하지 말 것:** 카톡·개인 메신저·개인 Gmail로 산출물 전달
- **반드시 할 것:** 암호는 **별도 채널**(문자/전화)로 전달
